# nodtime

**동료 6명의 회의 시간을, 근거와 함께 자동으로 확정하는 스케줄링 서비스**

같은 회사 동료 6명이 다음 주 안에 1시간 회의를 잡아야 합니다. 누군가는 외근·연차로 아예 불가능하고, 누군가는 "점심 직후는 피하고 싶다"는 정도의 비선호가 있습니다. 반드시 있어야 하는 필참자와, 있으면 좋은 선참자도 섞여 있습니다. nodtime은 이 서로 다른 성격의 조건들을 뒤섞지 않고 각자의 자리에 놓은 뒤, 순서대로 비교해 하나의 시간으로 좁혀가고 — 왜 그 시간인지, 왜 다른 시간은 아닌지까지 근거로 보여주는 회의 조율 경험입니다.

이 저장소는 실제 서비스 흐름을 그대로 재현한 화면 카탈로그(프로토타입)입니다. 좌측 화면 목록에서 항목을 고르면 우측에 실제 데이터와 인터랙션이 반영된 화면이 뜨고, 헤더의 라이트/다크 토글로 두 테마 모두 확인할 수 있습니다.

## 플로우별 설계 의도

### 1. 참여자 초대 — 조건보다 역할을 먼저 고정한다
동료를 추가하면서 필참·선참 토글로 역할부터 정합니다. 그 아래 "가능한 시간 한눈에 보기" 패널에서 전원의 요일별 가용성을 바로 볼 수 있는데, 이걸 조건 입력보다 먼저 배치한 이유는 — 회의 조율이 어려운 근본 원인이 "역할과 시간 조건이 뒤섞인 채로 판단이 시작되는 것"이라 보았기 때문입니다. 누가 필참인지가 먼저 고정돼야, 다음 단계의 모든 판단(정족수·우선순위)이 그 위에서 일관되게 굴러갑니다.

### 2. 회의 설정 — "성립 가능한가"와 "더 나은 시간은 무엇인가"를 화면 자체로 분리한다
좌측은 조건 입력(이름·기간·시간·최소 인원), 우측은 실시간 미리보기로 나눴습니다. 특히 최소 인원(정족수)은 필참 인원에 선참을 몇 명까지 더 요구할지를 "필참만 / 선참 N명 포함 / 전체 참석" 단계별 버튼으로 고르게 했는데, 이는 정족수가 필참 여부와는 별개로 회의 성격에 따라 조절돼야 하는 축이라고 판단했기 때문입니다. 우측 미리보기는 요일별 참석 가능 인원을 막대그래프로, 정족수 기준선을 점선으로 겹쳐 보여줘 — 저장하기 전에 "이 조건으로 회의가 성립하는지"를 계산 없이 눈으로 바로 확인할 수 있습니다.

### 3. 정족수 미충족 — 실패로 끝나는 화면을 만들지 않는다
조건상 회의가 성립하지 않으면 미리보기가 붉은 막대와 "성립 불가" 배지로 바뀌고, 곧바로 병목 원인 카드가 이어집니다. 여기엔 두 가지가 함께 있습니다: 누구의 어떤 일정이 막고 있는지("하린 — 요청 기간 전체 외근" 같은 구체적 사유), 그리고 실제로 성립을 만들어내는 해결 옵션만(정족수 하향, 기간 확장) 골라 제시하는 것. 성립되지 않는 옵션은 애초에 목록에 올리지 않아, 눌러도 아무것도 해결되지 않는 "죽은 버튼"이 없게 했습니다.

### 4. 최적 시간 찾는 중 — 판단 과정 자체를 화면으로 노출한다
로딩 화면은 단순한 스피너 대신, 실제 비교가 벌어지는 순서를 그대로 보여줍니다. 먼저 근무시간 외·점심시간 제외 → 필참 다수 불가 시간 제외로 후보를 추리고, 이후 ① 참석 인원 비교 → ② 필참자 비선호 회피 → ③ 선참자 참석 가능 여부 → ④ 동률이면 가장 이른 시간, 네 기준을 순서대로 적용합니다. 여러 조건을 하나의 점수로 합치지 않고 순차 비교로 설계한 이유는, 점수화하는 순간 "왜 이 시간인가"를 설명할 수 없게 되기 때문입니다. 필참 기준이 항상 선참보다 앞서는 것도 같은 이유로, 선참을 배려하다 필참 다수가 더 불편한 시간으로 밀리는 일을 막기 위한 장치입니다.

### 5. 추천 결과 — 결정뿐 아니라 포기한 부분까지 보여준다
결과 화면은 추천 일정과 "추천 이유"만 있는 게 아니라, 두 개의 상황 안내 배너를 따로 둡니다. 앰버 배너는 "민준은 점심 직후라 조금 불편하지만 참석은 가능해요"처럼 비선호를 참석 불가가 아닌 참고 정보로 남기고, 블루 배너는 "선참 유나는 겹치지만 필참을 우선하는 기준이라 그대로 추천했다"는 판단 근거를 그대로 밝힙니다. 아래 요일 비교·시간 비교 표는 선택되지 않은 후보들이 각각 어떤 기준에서 밀렸는지(13:00은 비선호와 겹침, 16:00은 선참 일정과 겹침 등) 보여주고, '대안 보기'를 열면 일부만 참석 가능한 케이스도 "누가 왜 빠지는지" 투명하게 밝힌 채로 진행할 수 있게 했습니다.

### 참여자 플로우 — 확인 이상의 입력을 요구하지 않는다
초대받은 사람은 마이페이지에 미리 등록해둔 비선호 시간이 자동으로 반영돼, 이 회의를 위해 같은 내용을 다시 입력할 필요가 없습니다. 참석자 목록 화면은 주최자의 "참여자 초대" 화면과 의도적으로 같은 구성(역할별 명단 + 전원 가용성 개요)을 읽기 전용으로 재사용해, 참여자가 새로운 화면을 학습할 필요 없이 같은 맥락으로 상황을 파악하게 했습니다.

### 설정 — 비선호는 가볍게, 캘린더 연동은 신뢰 있게
비선호 시간은 요일 × 오전/점심 후 오후/퇴근 전 오후 3구획 그리드에서 고르되 최대 3개로 제한했습니다. 비선호를 무제한으로 허용하면 사실상 모든 시간이 걸러져 정보로서 의미가 없어지기 때문입니다. 캘린더 연동 화면은 연동 여부를 상태로 명확히 보여주고, "근무시간(9~18시)만 반영·점심시간 자동 제외·확정 시 자동 등록"이라는 반영 범위를 명시해 추천 결과가 어떤 데이터로 만들어졌는지 신뢰할 수 있게 했습니다.

## 화면 구성

| 구분 | 화면 흐름 |
| --- | --- |
| 주최자 플로우 | 회의 목록 → 참여자 초대 → 회의 설정(실시간 미리보기 · 성립 실패 분기) → 최적 시간 찾는 중 → 추천 결과 |
| 참여자 플로우 | 초대 알림 → 참석자 목록 보기 → 회의 결과 보기 |
| 그 외 | 설정 허브 → 비선호 시간 설정 → 캘린더 연동 설정 |
